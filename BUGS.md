BUGS REPORT

BUG-001: В ответе на создание объявления поле status

Краткое описание: 
API не возвращает поле `id` при создании объявления и другие поля объявления, вместо этого возвращается строка в поле `status`.

Шаги воспроизведения:
1. Отправить POST `/api/1/item` с валидным телом
2. Посмотреть тело ответа

Фактический результат:  

{
  "status": "Сохранили объявление - 60ea3b2a-7e2d-4862-94d4-2a7d83ae87a9"
}

Ожидаемый результат:

{
  "id": "<string>",
  "sellerId": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": {
    "likes": "<integer>",
    "viewCount": "<integer>",
    "contacts": "<integer>"
  },
  "createdAt": "<string>"
}

Серьёзность: Major

Окружение:
API: {BASE_URL}/api/1/item
Метод: POST

BUG-002: Нулевые значения в статистике не принимаются при создании объявления

Краткое описание:
API возвращает ошибку 400 при попытке создать объявление с нулевыми значениями в полях likes, viewCount или contacts.

Шаги воспроизведения:

1. Отправить POST запрос на {BASE_URL}/api/1/item с данными:
Все поля валидны, кроме одного из полей статистики = 0
2.Проверить статус ответа

Фактический результат:
статус 400 Bad Request, объявление не создается

Ожидаемый результат:
статус 200 OK, объявление создается

Серьёзность: Minor

Окружение:
API: {BASE_URL}/api/1/item
Метод: POST

---------------

BUG-006: Статус ответа 405 при получении объявлений по sellerId с пустым sellerId
Краткое описание:
При запросе объявлений по SellerId с пустым sellerId API возвращает статус 405 Method Not Allowed вместо 400 Bad Request.

Шаги воспроизведения:

1.Отправить GET запрос на {BASE_URL}/api/1//item (двойной слеш, пустой sellerId)

Фактический результат:
HTTP статус 405 Method Not Allowed

Ожидаемый результат:
HTTP статус 400 Bad Request

Серьёзность: Minor

Окружение:
API: {BASE_URL}/api/1//item
Метод: GET
